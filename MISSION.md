# Mission: Terminal Addon Literacy

## Why
Learn what each terminal addon does so the shell feels predictable across Windows PowerShell, WSL Bash, Tabby, and Codex-adjacent terminals. The goal is to know which tool owns each behavior, so broken hotkeys or startup weirdness can be debugged without guesswork.

## Success looks like
- Identify whether history, fuzzy search, directory jumping, prompt rendering, or line editing is responsible for a terminal behavior.
- Explain what the Atuin, fzf, PSFzf, zoxide, Starship, Oh My Posh, and PSReadLine init lines do.
- Predict which key opens which UI in PowerShell and WSL Bash.
- Diagnose a missing command, broken key binding, or prompt issue from the right config file.

## Constraints
- Keep lessons short and directly tied to this workstation's actual config.
- Prefer official docs and local config evidence over generic terminal advice.
- Preserve existing shell behavior unless a lesson explicitly asks for a change.

## Out of scope
- Redesigning the terminal setup.
- Installing new shell frameworks.
- Deep shell scripting beyond what is needed to understand current addons.
