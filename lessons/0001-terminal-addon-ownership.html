Terminal Addon Ownership

Skill for this lesson: look at a terminal behavior and name the addon responsible. Once owner is known, debugging gets small.

Lesson 0001 Estimated time: 8 minutes Workspace: agents-toolkit

One Mental Model

Your terminal setup is not one tool. It is layers. Each layer owns one class of behavior.

Terminal appTabby/Windows Terminal shows panes, tabs, fonts, colors, key routing. It does not decide what Ctrl-R means inside a shell prompt.
ShellPowerShell or Bash runs commands and loads profile files. Addons become active only when profile init code runs.
Line editorPSReadLine owns PowerShell prompt editing. In WSL Bash, ble.sh is loaded and replaces plain GNU Readline as the interactive line editor.
Prompt rendererStarship or Oh My Posh draws the prompt: OS, shell, cwd, git, duration, time. It does not search history.
Workflow addonsAtuin, fzf, PSFzf, and zoxide add history search, fuzzy picking, and smart directory jumping.

Your Active Addons

Atuin: command history brain

Atuin replaces plain shell history with a SQLite-backed history store plus search UI. Official docs describe extra command context and optional end-to-end encrypted sync.

Your config: PowerShell runs atuin init powershell --disable-up-arrow --disable-ai. WSL Bash runs atuin init bash --disable-up-arrow. Result: Ctrl-R opens Atuin; up-arrow stays normal shell history.

Practical answer: Atuin can replace your day-to-day history search and recall workflow, but not literally erase the shell's own history layer. Bash still has history, HISTFILE, HISTSIZE, and .bash_history; PowerShell still has PSReadLine history unless separately changed. Atuin's own import docs also state old history files continue updating.

fzf: generic fuzzy picker

fzf is a filter UI: give it lines, type fuzzy text, select one or more lines. It is not specifically a history tool. It becomes file picker, directory picker, process picker, or git picker depending what feeds it input.

Your WSL Bash sources fzf completion and key-binding scripts. It also sets FZF_DEFAULT_COMMAND, FZF_CTRL_T_COMMAND, and FZF_ALT_C_COMMAND through fd so hidden files/directories are included while noisy dirs are excluded.

PSFzf: PowerShell-to-fzf bridge

PSFzf is not another picker. It wraps fzf for PowerShell and registers PSReadLine key handlers.

Your profile loads PSFzf only when fzf.exe exists, then sets Ctrl-T for provider/file selection and Alt-C for directory selection. Atuin loads after PSFzf, so history search belongs to Atuin in practice.

zoxide: smart cd memory

zoxide tracks directories you use and ranks them. z foo jumps to best remembered match. zi foo asks fzf when you want to choose interactively.

Your PowerShell and Bash profiles both eval zoxide init, which defines shell functions. Install alone is not enough because a child process cannot change parent shell cwd.

Starship: current main prompt

Starship draws a portable prompt from modules. Your shared config shows OS, shell, git branch/status, command duration, time, directory, and success/error prompt symbol.

PowerShell 7 sets STARSHIP_CONFIG to $HOME\.config\starship.toml. WSL Bash uses starship init bash.

Oh My Posh: legacy prompt

Oh My Posh still exists in Windows PowerShell 5.1 profile. It chooses a Codex-safe theme when it detects Codex host; otherwise clean detailed theme.

PowerShell 7 profile now uses Starship instead, so prompt issue in pwsh points to Starship, not Oh My Posh.

PSReadLine: PowerShell line editor

PSReadLine owns edit mode, bell, predictive suggestions, and Tab behavior. Microsoft docs define PredictionSource and PredictionViewStyle through Set-PSReadLineOption.

Your profile sets Windows edit mode, disables bell, enables history predictions in ListView when output is not redirected, and maps Tab to MenuComplete.

ble.sh: WSL Bash line editor

ble.sh is Bash Line Editor. Its README says it replaces default GNU Readline and adds syntax highlighting, enhanced completion, auto-complete, vim mode, status-line features, and history-share support.

Your WSL .bashrc sources $HOME/.local/share/blesh/ble.sh after readline-based fzf hooks and before Starship, zoxide, Atuin, and direnv. That makes ble.sh the Bash-side PSReadLine equivalent: Bash editing, completion behavior, suggestions, and syntax highlighting should be debugged as ble.sh first, not plain Readline first.

Exact Startup Order

ShellProfile SectionEffect
PowerShell 7 Microsoft.PowerShell_profile.ps1 lines 16-50 Starship first, PSReadLine next, PSFzf next, zoxide next, Atuin last.
WSL Bash /home/crunch/.bashrc lines 236-266 fzf readline hooks load first. Then ble.sh takes over Bash editing before Starship, zoxide, Atuin, and direnv.
Windows PowerShell 5.1 Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1 lines 3-38 Only Oh My Posh prompt setup is visible in current profile slice.
Conflict rule: last key binding usually wins. That is why Atuin loading after PSFzf matters for Ctrl-R. In WSL Bash, fzf readline hooks load first, then ble.sh becomes the PSReadLine-equivalent line editor underneath later zoxide/Atuin hooks.

Practice

Pick owner. Feedback appears immediately.

You press Ctrl-R and expect searchable command history. Who owns it?
You type z agents and jump to an old repo path. Who decided target directory?
In PowerShell, Ctrl-T opens a fuzzy file/provider picker. Which wrapper attached that to PSReadLine?
Your PowerShell 7 prompt shows git branch, time, and cwd. Who renders that prompt?
In WSL Bash, syntax highlighting and richer Tab behavior changed. Who owns that line-editing layer?
