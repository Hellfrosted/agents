# Visual Style

Prefer calm, dense, developer-readable artifacts. Avoid generic SaaS hero
layouts, violet gradients, ornamental blobs, and decorative cards that do not
carry information.

# Report Structure

Lead with the answer, then show maps, evidence, tradeoffs, and next decisions.
Use navigation when the artifact has four or more major sections.

# Diagram Rules

Use Mermaid for graph topology, SVG for custom explanatory geometry, and charts
for quantitative evidence. Keep diagrams readable at desktop and mobile widths.

# Asset Policy

Generated raster images are opt-in per report plan and should start early when
needed.

# Preferred Specialist Skills

Use Impeccable or comparable design/taste skills when available.
